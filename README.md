システム名：Infrastructure Risk Navigator（インフラ・リスク・ナビゲーター）1. コンセプト「静的な地図データ」に「動的な外部API」を掛け合わせ、点検すべき優先順位をAI/ロジックで自動判定する意思決定支援ツール。2. 技術スタック地図エンジン: Leaflet.jsデータ形式: GeoJSON (インフラ拠点データ)外部API:OpenWeatherMap API: リアルタイムの降水量・風速を取得。国土地理院 WMS: ハザードマップ（浸水・土砂災害）のレイヤー。ロジック: JavaScript による独自のリスクスコアリング演算。3. 主要機能とロジック（ここが「見せつけ」ポイント）① リアルタイム・リスクスコアリング各インフラ拠点（橋梁、法面、電柱など）に対して、以下の式で「今すぐ行くべきか」を計算する。$$RiskScore = (StaticRisk \times HazardWeight) + (RealtimeWeather \times CurrentImpact)$$StaticRisk: その設備が作られてからの年数（老朽化）。HazardWeight: ハザードマップ上の危険度（0〜1.0）。RealtimeWeather: 今、その場所で降っている雨量。② 優先度別カラーマーキング計算された RiskScore に基づき、地図上のピンの色を自動で変更する。赤（High）: 24時間以内に点検が必要。黄（Mid）: 1週間以内にスケジュールを組むべき。青（Low）: 異常なし。4. 画面仕様（UI/UX）メインマップ: 北九州全域のインフラ拠点を表示。レイヤー切り替えパネル:[ ] 洪水浸水想定区域（国土地理院）[ ] 土砂災害警戒区域（国土地理院）[ ] リアルタイム雨量雲（OpenWeatherMap）自動レポート機能:「優先度：高」の拠点をリストアップし、クリック一つで「点検指示書（ドラフト）」をテキスト生成する（克哉さんの得意なテキスト処理技術）。
